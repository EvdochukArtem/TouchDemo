# TouchDemo

Приложение для проработки взаимодействия с сенсорным дисплеем в кабине самолета.
В приложении реализован ввод данных с помощью сенсорных нажатий и/или клавиатуры
и мыши.

Доступные жесты:

-нажатие пальцом (нажате левой кнопкой мыши):
  взаимодействие с объектом, выделение объекта, нажатие на кнопку. Для выполнения
  действия необходимо, дождаться смыкания окружности символа индикации нажатия
  (крест с окружностью) прежде чем отпустить палец(кнопку мыши). При сильном сдвиге
  относительно первого касания жест будет распознан как перетягивание.
  
-прикосновение вторым пальцем при удержании нажатия первым (нажатие правой кнопки мыши):
  сброс нажатия, сброс объекта.
  
-перемещение пальца по экрану (перемещение зажатой левой кнопки мышки)Ж
  перетягивание объекта.
  
-жест Zoom(прокрутка колесика мышки):
  увеличение/уменьшение объекта
  
-поворот двумя пальцами(перемещение мышки с зажатой средней клавишей):
  поворот объекта
  
-свайп двумя пальцами(клавиши стрелок на клавиатуре):
  при начале жеста в одном кадре-четвертинке и окончании в другом - расширяет
  кадр до размера половинки. Свайп должен быть горизонтальный. Кадр в сторону которого
  происходит расширение либо сужается либо скрывается. Если жест начинается и
  заканчивается в пределах кадра осуществляется уменьшение размеров кадра до четверти
  (если жест горизонтальный и кадр был размером половинка) или до восьмушки (если
  кадр был четвертушкой и жест вертикальный снизу вверх). Для скрытия нижнего кадра-
  восьмушки применяется свайп из верхнего кадра в нижний.
  
Состав приложения:

-пакет util:
	содержит утилитарные классы обеспечивающие различные функции по управлению
	объектами, распознованию жестов и др. Все классы пакета должны иметь приватный
  конструктор и создаваться только классом Util. Класс Util реализует шаблон singleton
  и обеспечивает доступ к остальным утилитарным классам. Для сокращения обращения к
  утилитарному объекту через Util рекомендуется	использовать #define в StdAfx.h

-пакет object:
	содержит классы представляющие объекты с которыми будет взаимодействовать пользователь
	в процессе работы с приложением. В пакете есть базовые классы (Kadr, Frame, MechanicMenu)
	которые образуют базовый набор элементов в окне приложения. Объекты базовых классов
	должны содержать в себе остальные объекты и осуществлять взаимодействие с ними. Управление
	базовыми объектами и взаимодействие между ними должно осуществляется в EkranHandler.

-TouchDemo:
	точка входа в программы. Содержит глобальные переменные создает окно приложения и получает
  все поступающие сообщения от системы для приложения.
  
 Более подробная информация о предназначении классов описана header-файла класса.
